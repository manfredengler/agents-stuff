# Scoring

## Rúbrica de evaluación (1-5)

| Score | Criterio |
|-------|----------|
| 5 | Respuesta completa, precisa y con razonamiento propio. Sin errores conceptuales. |
| 4 | Respuesta correcta con alguna imprecisión menor o vocabulario aproximado. |
| 3 | Respuesta parcialmente correcta. Capta la idea central pero le faltan matices. |
| 2 | Respuesta con errores conceptuales importantes pero con algún elemento correcto. |
| 1 | Respuesta incorrecta, fuera de tema, o "no sé". |

**Umbral de avance:** score ≥ 3. Con score < 3 se repite (máx. 2 intentos) o se baja el nivel.

## Tipos de pregunta por nivel

### Novice

Preguntas de reconocimiento y definición:

- *"¿Qué es X?"*
- *"¿Para qué sirve X?"*
- *"¿Cuál es la diferencia entre X e Y?"*
- *"Completá: X es cuando…"*

Evitar: preguntas que requieran código, matemáticas o jerga avanzada.

### Intermediate

Preguntas de comprensión y aplicación:

- *"¿Cómo funciona X internamente?"*
- *"¿Cuándo usarías X en lugar de Y?"*
- *"¿Qué problema resuelve X?"*
- *"Describí el flujo de X en una aplicación real."*

### Advanced

Preguntas de análisis, diseño y trade-offs:

- *"¿Cuáles son las limitaciones de X?"*
- *"¿Cómo escalarías X para [escenario]?"*
- *"Compará X e Y en términos de [criterio técnico]."*
- *"¿Qué decisiones de diseño implica usar X?"*
- *"¿Cuándo X es la elección equivocada?"*

## Feedback según score

| Score | Feedback |
|-------|----------|
| 5 | Elogio directo + resumen del punto clave + avanzar. |
| 4 | Elogio + corrección de la imprecisión + avanzar. |
| 3 | Reconocer el acierto parcial + completar lo que faltó + avanzar. |
| 2 | Identificar el error sin juzgar + reformular el concepto + reintentar. |
| 1 | Dar una pista (no la respuesta) + reformular la pregunta + reintentar. |
