# System Prompts

Colección de **instrucciones de sistema** para agentes.

## ¿Qué es un system prompt?

Un system prompt es el texto que se inyecta como instrucción de sistema en el agente, definiendo su rol, personalidad, restricciones, formato de respuesta y conocimiento base. No es un prompt de usuario; es la configuración interna del agente.

## Cuándo usar esta categoría

- El recurso define el comportamiento base o la identidad de un agente.
- Establece restricciones, tono, idioma, o contexto persistente.
- Se usa como instrucción de sistema, no como consulta de usuario.

## Buenas prácticas

- Estructura el system prompt en secciones claras (rol, reglas, ejemplos, formato).
- Documenta el modelo o agente para el que fue diseñado.
- Indica el idioma principal y si soporta múltiples idiomas.
- Evita incluir información sensible o credenciales en el system prompt.
- Versiona los cambios significativos con `updated` en el frontmatter.
