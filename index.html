01

El catálogo

Visión general

Un repositorio donde cada creación tiene su propia carpeta, documentación y código listo para usar. Organizado por tipo para encontrar rápido lo que necesitás.

💡
Filosofía: cada recurso resuelve un problema concreto. Sin abstracciones innecesarias. Si encaja en tu flujo, lo copiás, lo importás y funciona.
Encontrás algo útil → Lo importás / clonás → Lo usás en tu agente → Listo
Estructura del repo github.com/manfredengler/agents-stuff 
agents-stuff/
├── catalogo/
│   ├── skills/          ← habilidades invocables por el agente
│   ├── plugins/         ← extensiones para VS Code / Copilot
│   ├── conectores/      ← integraciones con APIs externas
│   ├── prompts/         ← plantillas de prompt reutilizables
│   ├── system-prompts/  ← instrucciones de sistema para agentes
│   ├── hooks/           ← automatizaciones y manejadores de eventos
│   ├── mcp/             ← servidores Model Context Protocol
│   └── otros/           ← lo que no encaja en otra categoría
└── index.html           ← esta página
Convenciones de cada recurso 
catalogo/{categoria}/{nombre-del-recurso}/
├── README.md       ← documentación con frontmatter obligatorio
└── [archivos]      ← código, SKILL.md, referencias, ejemplos

Frontmatter obligatorio en README.md:
  title, description, category, author, tags, created, updated
02

Skills

1 disponible

Habilidades invocables por el agente. Cada skill define un comportamiento completo con su system prompt, flujos de trabajo y archivos de referencia cargados bajo demanda.

grill-me-concepts-with-memory
✅ estable claude.ai

Tutor socrático con memoria persistente. Detecta el nivel del usuario con una pregunta calibradora, explica conceptos con fuentes verificadas y interroga con exactamente 1 pregunta por turno — nunca da la respuesta directa. Guarda el progreso en un árbol jerárquico de markdowns (INDEX.md → topic.md → concepts/{slug}.md) con frontmatter.

/nivel /vibe /energia /next /retry /mapa /explorar /stats /fin
Qué genera en memoria /home/claude/learning/ 
INDEX.md                          ← mapa de todos los temas
topics/
└── {topic-slug}/
    ├── topic.md                  ← resumen del tema + tabla de conceptos
    └── concepts/
        └── {concept-slug}.md    ← concepto evaluado (score, attempts, fuentes)
Archivos del skill catalogo/skills/grill-me-concepts-with-memory/ 
SKILL.md            ← instrucciones principales (≤100 líneas)
references/
├── WORKFLOWS.md    ← flujos: calibración, interrogación, comandos, cierre
├── MEMORY-TREE.md  ← estructura del árbol de markdowns con frontmatter
├── SCORING.md      ← rúbrica 1-5 y tipos de pregunta por nivel
├── VIBE.md         ← guía de tono (serio / directo / irreverente) y energía
└── EXPLORER.md     ← spec del dashboard HTML generado por /explorar
⚡
Dashboard /explorar: genera un HTML autocontenido con visualización interactiva de toda la memoria — árbol de temas, distribución de scores, filtros por estado. Sin dependencias externas.
03

Categorías

8 tipos

El catálogo cubre todos los tipos de recursos que un agente puede necesitar. Cada categoría tiene su propio README con reglas de cuándo usarla.

⚡
skills/ 1

Habilidades invocables. Definen un comportamiento completo con system prompt, flujos y archivos de referencia.

🔌
plugins/ 0

Extensiones para VS Code y GitHub Copilot. Amplían las capacidades del editor con herramientas personalizadas.

🔗
conectores/ 0

Integraciones con APIs y fuentes de datos externas. El agente puede consultarlas mediante tool calls.

📝
prompts/ 0

Plantillas de prompt orientadas al usuario final. Reutilizables en distintos contextos y agentes.

🤖
system-prompts/ 0

Instrucciones de sistema que definen el rol, contexto y restricciones de un agente desde el inicio.

🪝
hooks/ 0

Manejadores de eventos y automatizaciones. Se ejecutan en respuesta a acciones del agente (PreToolUse, PostToolUse, Stop).

⚙️
mcp/ 0

Servidores y herramientas Model Context Protocol. Exponen capacidades externas como tools nativos.

📦
otros/ 0

Recursos que no encajan en ninguna otra categoría. Útil para experimentos y creaciones inclasificables.

🎯
¿Dudás en qué categoría va algo? Elegí la más específica. Si el recurso define un comportamiento invocable → skills/. Si se conecta a una API → conectores/. Si no encajá en nada → otros/ y abrí una issue.
04

Cómo usar

Importar · Contribuir

Tres formas de usar lo que hay en el catálogo: importar directamente desde GitHub, clonar el repo, o contribuir con tus propias creaciones.

Opción 1 — Importar un skill en Claude.ai
1

Abrí el SKILL.md del skill que querés usar en GitHub.

2

Hacé click en Raw para obtener la URL del contenido crudo.

3

En Claude.ai, creá un nuevo proyecto y pegá el contenido del SKILL.md como instrucción de sistema.

Opción 2 — Clonar el repo completo 
git clone https://github.com/manfredengler/agents-stuff.git
cd agents-stuff

# Explorar los skills disponibles
ls catalogo/skills/

# Ver detalle de un skill
cat catalogo/skills/grill-me-concepts-with-memory/README.md
Opción 3 — Contribuir un recurso 
# 1. Crear carpeta para tu recurso
mkdir -p catalogo/skills/mi-skill

# 2. Añadir README.md con frontmatter obligatorio
# title, description, category, author, tags, created, updated

# 3. Añadir los archivos del recurso (SKILL.md, referencias, etc.)

# 4. Añadir entrada en la tabla de la categoría correspondiente
# catalogo/skills/README.md → tabla de entradas

# 5. Abrir PR
gh pr create
🔒
Regla de calidad: el SKILL.md debe tener menos de 100 líneas. El contenido extenso va en references/ y se carga bajo demanda. Esto mantiene el contexto del agente limpio y predecible.
